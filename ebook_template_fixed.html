Chapter 1

為什麼要做成網頁版電子書

不是做一個普通網站，而是做一個可以長時間閱讀的介面。

很多人把電子書網站做成首頁式排版，畫面很漂亮，但一旦文章變長，閱讀體驗就開始下降。真正好的網頁版電子書，重點不是炫，而是讓讀者願意一直看下去。

因此，這份模板把焦點放在三件事：第一是文字閱讀的舒適度，第二是章節結構的清楚程度，第三是技術內容的呈現品質。尤其如果您的內容有程式碼、表格、重點框，版型就更需要穩定。

重點提醒 電子書頁面應該優先考慮可讀性，而不是一開始就追求大量動態效果。
好的閱讀介面，不是讓讀者注意到設計，而是讓讀者忘記設計的存在。
Chapter 2

閱讀介面的關鍵原則

字級、行高、欄寬、留白，都會直接影響讀者是否想繼續看。

1. 文字欄寬不要太寬

長文如果鋪滿整個桌面螢幕，眼睛來回掃描會很累。這個模板已經把主要內文寬度控制在適合閱讀的範圍內。

2. 標題階層要明顯

章節名稱、小節名稱、內文段落，必須有清楚的視覺差異。這不只是美觀問題，也是理解效率問題。

教學內容建議 如果您的書是教學型內容，可以固定每節都有：概念說明、範例、常見錯誤、練習題。讀者會更容易建立閱讀節奏。

3. 表格與整理區要一致

元素 用途 建議
章節卡片 區分內容區塊 保持圓角、邊框、陰影一致
重點框 提示重要內容 用淡色背景，不要太刺眼
程式碼區塊 顯示範例程式 深色底、可橫向捲動、可複製
Chapter 3

程式教學內容怎麼排

技術書最常失敗的地方，就是程式碼好擠、說明太散、範例不集中。

如果是資訊教材，建議每個小節都用一致節奏呈現，例如：先講概念，再放程式碼，再補說明，最後給一題小練習。這樣讀者會知道每一節怎麼讀，理解也會更穩定。

範例：簡單 JavaScript 互動

const button = document.querySelector('#themeToggle');
button.addEventListener('click', () => {
  document.body.classList.toggle('dark');
});
練習題 請把上方按鈕功能改成「切換字體大小」，並思考：為什麼閱讀介面通常需要字體大小切換功能？

常見錯誤

  • 一個小節塞太多程式碼，讀者不知重點在哪。
  • 程式碼與文字說明距離太遠，來回對照很累。
  • 所有提醒都用同一種框，視覺上沒有優先順序。
Chapter 4

如何快速替換成自己的內容

先固定模板，再批量替換章節，效率最高。

您只要改這些地方

  1. 修改頁面標題與書名。
  2. 把左側目錄連結改成您的章節。
  3. 複製一份 <article class="chapter"> 區塊，作為新章節。
  4. 把示意內容換成您的正文、程式碼、表格與練習。

適合再擴充的功能

  • 全文搜尋
  • 章節折疊
  • 字體切換
  • 註解功能
  • 多頁章節拆分
實務建議 如果您之後要交給 Codex 大量產生章節，請先把這份模板當成母版，再要求它「依母版格式新增章節」，會比每次重做整頁穩定很多。
