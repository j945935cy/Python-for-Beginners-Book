# Textbook Style Guide

Use this guide for the Python beginner book and similar Markdown textbooks.

## Audience

- First-time Python learners
- High school students
- College students
- Non-CS self learners

## Writing Style

- Use Traditional Chinese.
- Prefer Taiwan terminology.
- Keep paragraphs short.
- Explain by teaching, not by theory-first exposition.
- Introduce only one new idea at a time.
- Keep examples executable with Python 3.

## Scope Limit

The book stops at functions. Do not introduce these topics:

- class
- module/import as a new topic
- decorator
- generator
- inheritance
- package management
- advanced typing

## Chapter Sections

Every chapter should include:

1. 本章重點
2. 概念說明
3. 範例程式
4. 程式解說
5. 常見錯誤
6. 練習題
7. 小結

## Markdown and Code

- Use `#`, `##`, and `###` heading levels.
- Use fenced code blocks.
- Mark Python code blocks as `python`.
- Avoid raw HTML unless the repo already requires it.
- Use clear variable names.
- Keep each example focused on one concept.
