# 出版前校稿清單

這份清單適合在準備正式發布網站版、EPUB 版，或交付讀者試讀前使用。

## 一、內容正確性
- 書名、副標題、作者名稱是否在首頁、HTML、EPUB metadata 中一致。
- 各章標題與目錄名稱是否一致。
- 章節順序是否正確，沒有跳號或重複編號。
- 全書內容是否確實只寫到函式，沒有混入 class、module、decorator、generator 等超出範圍主題。
- 每一章的觀念說明是否符合初學者程度，沒有突然跳入進階語法。
- 每一章的例子是否只聚焦一個核心概念，沒有一次塞太多新東西。
- 程式術語是否一致，例如清單、字典、字串、函式、條件判斷、迴圈等。
- 英文術語若有保留，是否都有中文說明或第一次出現時有交代。

## 二、章節完整性
- 每一章是否都有以下固定段落：
  - 本章重點
  - 概念說明
  - 範例程式
  - 程式解說
  - 常見錯誤
  - 練習題
  - 小結
- 練習題參考解答與解析是否完整。
- 每一章開頭是否清楚說明本章要學什麼。
- 每一章結尾是否有自然銜接到下一章。

## 三、範例程式檢查
- 所有 Python 程式碼是否可執行。
- 變數名稱是否清楚、容易理解。
- 程式碼縮排是否正確。
- 輸出內容是否和解說一致。
- 使用者輸入的範例是否有清楚說明輸入資料型態。
- 需要轉型的地方是否有明確處理，例如 `int(input(...))`。
- 常見錯誤示範是否真的能對應到初學者常見錯誤。

## 四、教學節奏檢查
- 從第 1 章到第 10 章，難度是否逐步上升。
- 新觀念出現前，是否有使用前面已學過的知識做鋪陳。
- 同一章中，範例順序是否由簡單到稍微複雜。
- 練習題難度是否和本章內容相符。
- 小結是否有真正幫讀者收斂重點，而不是只是重複前文。

## 五、插圖與版面檢查
- 插圖是否和章節內容真的有關，沒有只是裝飾用途。
- 插圖標題或替代文字是否清楚。
- HTML 版插圖是否正常顯示。
- EPUB 版插圖是否正常嵌入。
- 插圖風格是否一致，沒有尺寸差太大或視覺風格落差太大。
- 程式碼區塊、標題、段落間距是否閱讀舒適。

## 六、Markdown 與輸出檢查
- Markdown 標題階層是否正確使用 `#`、`##`、`###`。
- 程式碼區塊是否都有 fenced code block。
- Python 程式碼區塊是否都有標註 `python`。
- 是否避免直接硬寫 HTML。
- `docs/index.html` 是否能正常作為 GitHub Pages 首頁。
- `output/html/index.html` 是否成功產出。
- `output/epub/python-for-beginners.epub` 是否成功產出。

## 七、GitHub Pages 與發布檢查
- `main` 分支是否已推上最新版本。
- `docs/` 內容是否為最新建置結果。
- GitHub Pages 設定是否指向正確來源。
- 首頁標題、封面、目錄、插圖是否在線上版本正常顯示。
- 行動裝置與桌面版瀏覽時，排版是否都可閱讀。

## 八、最後人工閱讀檢查
- 以「完全沒學過 Python 的讀者」角度，從首頁一路讀到第 3 章，看是否有難懂或跳太快的地方。
- 抽查至少 3 章，把範例程式親手重打一次，確認教學流程合理。
- 抽查至少 2 章的練習題，確認題目沒有超出本章範圍。
- 檢查是否有錯字、標點不一致、大小寫不一致、章名不一致。
- 確認整本書的語氣都維持白話、教學導向、適合第一次學 Python 的讀者。
